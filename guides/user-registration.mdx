## Overview

FlushLa uses a digital pass system integrated with Apple Wallet to manage user authentication and access. This guide walks you through the registration and login process.

## Creating an Account

<Steps>
  <Step title="Navigate to Registration">
    Visit `/register` or click "Create one" from the login page.
  </Step>

  <Step title="Fill Registration Form">
    Enter the following information:
    - **Full Name**: Your display name in the system
    - **Email**: A valid email address where you'll receive your access pass

    <Note>
      The email is used for both authentication and receiving your Apple Wallet pass.
    </Note>
  </Step>

  <Step title="Submit Registration">
    Click **Create Account** to submit your registration.

    The system will:
    1. Create a Supabase Auth user account via `/api/create_passkey`
    2. Generate a user profile in the database with an initial score of 41
    3. Create a digital pass through PassCreator API
    4. Send the pass to your email
  </Step>

  <Step title="Add Pass to Apple Wallet">
    Check your email for the Apple Wallet pass. Tap to add it to your wallet.

    <Info>
      Your pass displays your cleanliness score and updates automatically after each inspection.
    </Info>
  </Step>
</Steps>

## Logging In

<Steps>
  <Step title="Go to Login Page">
    Navigate to `/login` or click the login link from the landing page.
  </Step>

  <Step title="Enter Credentials">
    Provide your:
    - **Username**: Your email address used during registration
    - **Password**: Default is `1234` (should be changed in production)

    <Warning>
      The current implementation uses a default password. In production, users should set custom passwords.
    </Warning>
  </Step>

  <Step title="Submit Login">
    Click **Login** to authenticate.

    The system calls `/api/login` which:
    - Authenticates via Supabase Auth with `sign_in_with_password()`
    - Fetches your user profile from the database
    - Creates a session with `user_id` stored in session cookies
  </Step>

  <Step title="Access Dashboard">
    Upon successful login, you'll be redirected to:
    - **User Dashboard** (`/`) if you're a regular user
    - **Admin Dashboard** (`/admin`) if you have admin privileges
  </Step>
</Steps>

## Session Management

### Active Session

Once logged in, your session persists through:
- **Session Middleware**: FastAPI's `SessionMiddleware` with secret key `"motherwaterplant"`
- **Session Storage**: User ID stored in `request.session["user_id"]`

### Checking Current User

The `get_current_user()` helper function:
1. Retrieves `user_id` from session
2. Queries the `users` table in Supabase
3. Returns user data including username, email, score, and admin status

### Logging Out

To log out:

<Steps>
  <Step title="Access Logout Endpoint">
    Navigate to `/api/logout` or click the logout button in your dashboard.
  </Step>

  <Step title="Session Cleared">
    The system will:
    - Clear all session data with `request.session.clear()`
    - Sign out from Supabase Auth with `supabase.auth.sign_out()`
    - Redirect to `/landing`
  </Step>
</Steps>

## User Tiers

Your account is automatically assigned a tier based on your cleanliness score:

| Tier | Score Range | Tier Name | Access Level |
|------|-------------|-----------|-------------|
| 3 | 80-100 | Gold Standard | All toilets |
| 2 | 50-79 | Silver Standard | Silver & Bronze toilets |
| 1 | 0-49 | Bronze Standard | Bronze toilets only |

<Tip>
  Maintain high cleanliness scores during inspections to unlock access to higher-tier facilities.
</Tip>

## Related Endpoints

- **POST** `/api/create_passkey` - Create new user account and digital pass
- **POST** `/api/login` - Authenticate user and create session
- **GET** `/api/logout` - Clear session and sign out
- **GET** `/register` - Registration page
- **GET** `/login` - Login page
- **GET** `/` - User dashboard (requires authentication)

## Troubleshooting

<AccordionGroup>
  <Accordion title="Account already exists error">
    This means the email is already registered. Use the login page instead or contact support if you believe this is an error.
  </Accordion>

  <Accordion title="Invalid credentials during login">
    Verify you're using the correct email address. The default password is `1234` unless you've changed it.
  </Accordion>

  <Accordion title="Didn't receive Apple Wallet pass">
    Check your spam folder. The pass is sent via PassCreator API. If still not received, try registering again or contact support.
  </Accordion>

  <Accordion title="Session expired">
    Sessions may expire. Simply log in again to create a new session.
  </Accordion>
</AccordionGroup>
