## Overview

The Nearby Toilets feature helps you find available toilet facilities, view their cleanliness ratings, check access requirements, and plan your visits based on your tier level.

## Accessing the Page

Navigate to `/nearby-toilets` from:
- Your user dashboard navigation menu
- Direct URL access

<Note>
  You must be logged in to access this page. Unauthenticated users are redirected to `/login`.
</Note>

## Filtering Toilets

Use the filter bar to view specific tiers:

```javascript
function filterTier(tier) {
  const cards = document.querySelectorAll('.toilet-card');
  cards.forEach(card => {
    const cardTier = parseInt(card.dataset.tier);
    if (tier === 'all' || cardTier === tier) {
      card.style.display = 'block';
    } else {
      card.style.display = 'none';
    }
  });
}
```

## Viewing Cleanliness Ratings

### How Ratings Are Calculated

The backend (main.py:591) fetches toilet data with latest inspections:

```python
# Fetch all toilets
toilets_res = supabase
  .table("toilets")
  .select("*")
  .order("name")
  .execute()

# For each toilet, get latest inspection
for toilet in toilets:
    latest_visit_res = supabase
      .table("toilet_visits")
      .select("cleanliness_score, created_at, image_url")
      .eq("toilet_id", toilet["id"])
      .order("created_at", desc=True)
      .limit(1)
      .execute()
    
    toilet["latest_inspection"] = latest_visit_res.data[0] if latest_visit_res.data else None
```

### Rating Display

- **Score Badge**: Large number in bottom-left of card
- **Color Coding**: Visual indicator of cleanliness level
- **Last Inspected**: Timestamp of most recent inspection
- **No Inspections**: Gray text indicating no recent data

## Access Requirements

Your access to each toilet is determined by tier comparison:

```jinja
{% set user_tier = 3 if user.score >= 80 else (2 if user.score >= 50 else 1) %}
{% set can_access = user_tier >= toilet.current_tier %}
```

### Access Granted (Green Inspect Button)

When `user_tier >= toilet.current_tier`:


Hover to see:

## Backend Implementation

The `/nearby-toilets` endpoint (main.py:591):

<Steps>

  </Step>

  </Step>
</Steps>

## Database Schema

### Toilets Table

```sql
CREATE TABLE toilets (
  id SERIAL PRIMARY KEY,
  name TEXT NOT NULL,
  location TEXT,
  current_tier INTEGER DEFAULT 1,
  last_cleanliness_score INTEGER
);
```

### Toilet Visits Table

```sql
CREATE TABLE toilet_visits (
  id SERIAL PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  toilet_id INTEGER REFERENCES toilets(id),
  cleanliness_score INTEGER,
  score_delta INTEGER,
  image_url TEXT,
  ai_result JSONB,
  created_at TIMESTAMP
);
```

## Improving Your Access

To unlock higher-tier toilets:

<Steps>
  <Step title="Check Current Score">
    View your score in the top navigation or user profile.
  </Step>

  <Step title="Calculate Points Needed">
    - **Bronze → Silver**: Need 50 points
    - **Silver → Gold**: Need 80 points
  </Step>

  <Step title="Inspect Clean Toilets">
    Focus on inspecting well-maintained facilities:
    - **90-100 score**: +5 points
    - **70-89 score**: 0 points (neutral)
    - **Below 70**: Negative points
  </Step>

  <Step title="Maintain Consistency">
    Regular inspections of clean facilities gradually improve your score.
  </Step>
</Steps>

<Tip>
  Target Bronze tier toilets with high cleanliness scores to safely earn points without risking penalties.
</Tip>

## Troubleshooting

<AccordionGroup>
  <Accordion title="No toilets showing">
    **Possible causes**:
    - Database connection issue
    - No toilets in database
    - Filter excluding all results
    
    **Solution**: Try "All" filter or contact support.
  </Accordion>

  <Accordion title="Cleanliness scores not displaying">
    This means the toilet has no recent inspections. The score will appear once someone inspects it.
  </Accordion>

  <Accordion title="Can't access any toilets">
    Your score may be too low. Start with Bronze tier facilities (orange badge) which have minimal requirements.
  </Accordion>

  <Accordion title="Stats not updating after filtering">
    Refresh the page. The JavaScript `updateStats()` function should run automatically but may have encountered an error.
  </Accordion>
</AccordionGroup>
