## Endpoint

```
POST /api/create_passkey
```

## Overview

Registers a new user in the FlushLa system and creates a digital pass (wallet card) for them. This endpoint:

1. Validates that the email is not already registered
2. Creates a Supabase Auth user (with email verification bypassed)
3. Inserts user profile into the database
4. Creates a digital pass via PassCreator API
5. Stores the pass identifier
6. Automatically logs in the user

**Location**: `app/main.py:1055`

## Request Body

<ParamField body="templateId" type="string" required>

## Default User Properties

New users are created with:

- **Password**: `"1234"` (should be changed by user)
- **Score**: `41` (initial cleanliness score)
- **Email Confirmation**: Bypassed (set to confirmed)
- **Admin Status**: `false`

## PassCreator Integration

The endpoint calls the PassCreator API to generate a digital wallet pass:

```javascript
POST https://app.passcreator.com/api/v3/pass

Headers:
  Authorization: [PASSKEYAPI env variable]
  Content-Type: application/json

Body:
{
  "data": {
    "templateId": "template_abc123",
    "emailRecipient": "john.doe@example.com",
    "firstname": "John"
  }
}
```

The pass is automatically sent to the user's email and can be added to Apple Wallet or Google Pay.

## Session Management

Upon successful registration, the user is automatically logged in:

```python
request.session["user_id"] = user_id
```

The session cookie is sent to the client and can be used for subsequent authenticated requests.

## Error Handling

### Account Already Exists

**Status Code**: `200` (with error flag)

```json
{
  "error": true,
  "message": "Account already exists"
}
```

### PassCreator API Failure

**Status Code**: `200` (with error details)

```json
{
  "error": true,
  "status": 400,
  "raw": "PassCreator API error message"
}
```

### Invalid Pass Response

**Status Code**: `500`

```json
{
  "detail": "Invalid pass API response"
}
```
