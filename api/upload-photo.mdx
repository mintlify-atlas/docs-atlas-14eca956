## Endpoint

```
POST /upload-photo
```

## Overview

Uploads a toilet photo for AI analysis. When associated with a workflow session, this endpoint:

1. Receives the image file
2. Analyzes it using computer vision AI
3. Detects cleanliness issues
4. Calculates cleanliness score
5. Updates user score
6. Stores results in database
7. Returns annotated image with detections

**Location**: `app/main.py:80`

## Request Parameters

<ParamField body="image" type="file" required>
  The image file to upload (multipart/form-data)
</ParamField>

<ParamField body="session_id" type="string">
  Optional workflow session ID. If provided, triggers automatic analysis and score updates.
</ParamField>

## Content Type

```
Content-Type: multipart/form-data
```

## Response (With Session)

<ResponseField name="status" type="string">
  `"analysis_completed"` when analysis succeeds
</ResponseField>

<ResponseField name="session_id" type="string">
  The workflow session ID
</ResponseField>

<ResponseField name="analysis" type="object">
  AI analysis results including detected issues and cleanliness score
</ResponseField>

<ResponseField name="analysis.cleanliness_score" type="integer">
  Overall cleanliness score (0-100)
</ResponseField>

<ResponseField name="analysis.detections" type="array">
  Array of detected objects/issues with bounding boxes
</ResponseField>

<ResponseField name="annotated_image_base64" type="string">
  Base64-encoded image with bounding boxes drawn around detections
</ResponseField>

<ResponseField name="score_delta" type="integer">
  Points added/subtracted from user's score based on cleanliness
</ResponseField>

<ResponseField name="image_url" type="string">
  Public URL of the annotated image stored in Supabase Storage
</ResponseField>

## Response (Without Session)

<ResponseField name="status" type="string">
  `"photo_received"`
</ResponseField>

<ResponseField name="session_id" type="string">
  The session ID provided (or null)
</ResponseField>

## Score Delta Calculation

The `score_delta` is calculated based on cleanliness score:

| Cleanliness Score | Score Delta | Meaning |
|-------------------|-------------|----------|
| 90-100 | +5 | Bonus for very clean |
| 70-89 | 0 | Neutral |
| 50-69 | -5 | Small penalty |
| 30-49 | -10 | Medium penalty |
| 0-29 | -15 | Large penalty for very dirty |

## Image Storage

Annotated images are uploaded to Supabase Storage:

- **Bucket**: `uploads`
- **Filename format**: `toilet_inspection_{user_id}_{timestamp}.jpg`
- **Timezone**: Asia/Singapore (UTC+8)

**Implementation**: `app/main.py:177`

## Workflow Integration

This endpoint is designed to work with the [Start Inspection](/api/start-inspection) workflow:

1. Call `/api/start-toilet-inspection` to get a `session_id`
2. Upload photo to `/upload-photo` with the `session_id`
3. Analysis happens automatically
4. User score is updated
5. Results are stored in database
