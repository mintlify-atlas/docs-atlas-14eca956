## Endpoint

```
GET /user
```

<Note>
This is a web route that returns an HTML page, not a JSON API endpoint. For API access to user data, use the `get_current_user()` function.
</Note>

## Overview

Returns the authenticated user's profile page with their recent toilet inspection history.

**Location**: `app/main.py:537` (Home route)

## Authentication

Requires an active session. Redirects to landing page if not authenticated.

## Data Returned

The user profile page displays:

### User Information

- Username
- Email
- Current score (0-100)
- User tier (Bronze/Silver/Gold)
- Admin status

## User Tier System

User tiers are calculated based on score:

| Score Range | Tier | Name |
|-------------|------|------|
| 80-100 | 3 | Gold Standard |
| 50-79 | 2 | Silver Standard |
| 0-49 | 1 | Bronze Standard |

**Implementation**: `app/main.py:241`

```python
def get_user_tier(score: int) -> int:
    if score >= 80:
        return 3  # Gold
    elif score >= 50:
        return 2  # Silver
    else:
        return 1  # Bronze
```

## Score Updates

User scores are updated automatically after each toilet inspection via the `update_user_score()` function:

**Location**: `app/main.py:274`

### Update Process

1. Fetch the latest `score_delta` from `toilet_visits`
2. Get current user score
3. Calculate new score (clamped to 0-100)
4. Update database
5. Update digital pass via PassCreator API
6. Send push notification to user's wallet

### Score Bounds

```python
new_score = current_score + score_delta
if new_score < 0:
    new_score = 0
elif new_score > 100:
    new_score = 100
```

## PassCreator Integration

When a user's score changes, their digital wallet pass is updated:

```javascript
PATCH https://app.passcreator.com/api/v3/pass/{identifier}

Body:
{
  "data": {
    "storedValue": 75
  }
}
```

A push notification is also sent:

```javascript
POST https://app.passcreator.com/api/pass/{identifier}/sendpushnotification

Body:
{
  "pushNotificationText": "Your toilet cleanliness score has been updated to 75."
}
```

## Admin Users

Users with `is_admin: true` are redirected to the admin dashboard at `/admin` instead of the user profile page.
