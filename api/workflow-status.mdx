## Endpoint

```
GET /api/workflow-status/{session_id}
```

## Overview

Returns the current status of a workflow session, including analysis results when available.

**Location**: `app/main.py:396`

## Path Parameters

<ParamField path="session_id" type="string" required>
  The unique UUID of the workflow session
</ParamField>

## Response

<ResponseField name="session_id" type="string">
  The workflow session UUID
</ResponseField>

<ResponseField name="user_id" type="string">
  ID of the user who initiated the inspection
</ResponseField>

<ResponseField name="toilet_id" type="integer">
  ID of the toilet being inspected
</ResponseField>

<ResponseField name="status" type="string">
  Current workflow state (see table below)
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the session was created
</ResponseField>

<ResponseField name="analysis_result" type="object">
  Only present when `status` is `"completed"`. Contains full analysis data.
</ResponseField>

<ResponseField name="analysis_result.analysis" type="object">
  AI analysis results with cleanliness score and detections
</ResponseField>

<ResponseField name="analysis_result.annotated_image_base64" type="string">
  Base64-encoded annotated image with bounding boxes
</ResponseField>

<ResponseField name="analysis_result.score_delta" type="integer">
  Points change for the user's score
</ResponseField>

<ResponseField name="analysis_result.image_url" type="string">
  Public URL of the stored annotated image
</ResponseField>

## Session States

| Status | Description |
|--------|-------------|
| `waiting_for_capture` | Session created, waiting for camera to capture photo |
| `capture_triggered` | Camera has been signaled to take a photo |
| `analyzing` | Photo uploaded, AI analysis in progress |
| `completed` | Analysis complete, results available in `analysis_result` |
| `error` | An error occurred, check `analysis_result.error` |

## Polling Pattern

For real-time updates, implement polling:

```javascript
async function pollWorkflowStatus(sessionId) {
  const maxAttempts = 30;
  const pollInterval = 2000; // 2 seconds
  
  for (let i = 0; i < maxAttempts; i++) {
    const response = await fetch(`/api/workflow-status/${sessionId}`);
    const data = await response.json();
    
    if (data.status === 'completed' || data.status === 'error') {
      return data;
    }
    
    await new Promise(resolve => setTimeout(resolve, pollInterval));
  }
  
  throw new Error('Workflow timeout');
}
```

## Usage in Workflow

1. Start inspection: `POST /api/start-toilet-inspection`
2. Get `session_id` from response
3. Upload photo: `POST /upload-photo` with `session_id`
4. Poll this endpoint until `status` is `"completed"` or `"error"`
5. Retrieve analysis results from `analysis_result`
