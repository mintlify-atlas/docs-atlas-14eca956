## Endpoint

```
POST /api/login
```

## Overview

Authenticates a user using email and password, creates a session, and returns user profile information.

**Location**: `app/main.py:848`

## Request Body

<ParamField body="username" type="string" required>
  User's email address (despite the field name, this is the email)
</ParamField>

<ParamField body="password" type="string" required>
  User's password
</ParamField>

### Request Schema

```python
class LoginRequest(BaseModel):
    username: str  # Actually the email
    password: str
```

## Response (Success)

<ResponseField name="success" type="boolean">
  Always `true` on successful login
</ResponseField>

<ResponseField name="user_id" type="string">
  The authenticated user's UUID
</ResponseField>

<ResponseField name="username" type="string">
  User's display name
</ResponseField>

<ResponseField name="score" type="integer">
  User's current cleanliness score (0-100)
</ResponseField>

<ResponseField name="is_admin" type="boolean">
  Whether the user has admin privileges
</ResponseField>

## Response (Error)

<ResponseField name="error" type="boolean">
  `true` when authentication fails
</ResponseField>

<ResponseField name="message" type="string">
  Error description
</ResponseField>

## Example Request

```bash
curl -X POST http://localhost:8000/api/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "john.doe@example.com",
    "password": "securepassword123"
  }'
```

## Authentication Flow

1. **Supabase Auth Validation**
   ```python
   auth_response = supabase.auth.sign_in_with_password({
       "email": payload.username,
       "password": payload.password
   })
   ```

2. **Fetch User Profile**
   ```python
   res = supabase.table("users")
       .select("*")
       .eq("id", user_id)
       .single()
       .execute()
   ```

3. **Create Session**
   ```python
   request.session["user_id"] = user_id
   ```

4. **Return User Data**

## Session Cookie

Upon successful login, a session cookie is set:

- **Name**: Session ID (generated by SessionMiddleware)
- **HttpOnly**: Yes (prevents JavaScript access)
- **Secure**: Depends on HTTPS configuration
- **SameSite**: Default (Lax)

The session cookie is automatically sent with subsequent requests.

## Error Scenarios

### Invalid Email or Password

**Status Code**: `200` (with error flag)

```json
{
  "error": true,
  "message": "Invalid email or password"
}
```

This error is returned when:
- Email doesn't exist in Supabase Auth
- Password is incorrect
- Account is disabled

### User Profile Not Found

**Status Code**: `200` (with error flag)

```json
{
  "error": true,
  "message": "User profile not found"
}
```

This occurs when the Auth user exists but the corresponding profile in `public.users` is missing.

## Usage Example (JavaScript)

```javascript
async function login(email, password) {
  const response = await fetch('/api/login', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      username: email,  // Field name is 'username' but pass email
      password: password
    }),
    credentials: 'include'  // Important: include cookies
  });
  
  const data = await response.json();
  
  if (data.error) {
    console.error('Login failed:', data.message);
    return null;
  }
  
  console.log('Logged in as:', data.username);
  return data;
}
```

## Logout

To log out, call the [logout endpoint](/api/authentication#4-logout):

```bash
GET /api/logout
```

This will:
1. Clear the session
2. Sign out from Supabase Auth
3. Redirect to the landing page
